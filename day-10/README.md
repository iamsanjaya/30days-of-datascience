## What This Day Covers

### Standard Task

Train a Decision Tree (depth 4) on the Telco churn dataset, visualize the full tree, explain why top-of-tree features matter, then train a Random Forest and compare both with 5-fold Stratified CV.

### Out-of-Box Challenge — "Build the Tree in Your Head"

Manually trace the decision path for 10 rows using `decision_path()`. Find the most populated leaf and surface the most dissimilar pair of customers inside it — both scored identically, despite being very different people. Extend the RF analysis with OOB error stability and feature importance variance across random seeds.

## Results

### 5-Fold Stratified CV (Accuracy)

| Model                              | Mean       | Std                |
| ---------------------------------- | ---------- | ------------------ |
| Decision Tree (depth 4)            | ~0.787     | ~0.012             |
| Random Forest (200 trees, depth 8) | ~0.803     | ~0.007             |
| **RF improvement**                 | **+~1.6%** | **lower variance** |

### Key Findings

**Root split feature:** `Contract`
The single variable that divides churn vs no-churn most cleanly across all 7,043 customers. Month-to-month contract holders are far more likely to churn — the tree discovers this first.

**Leaf flattening problem**
The most populated leaf contained customers that differed by large amounts on `tenure` and `MonthlyCharges`, yet both received the exact same `P(Churn)`. Once the tree has exhausted its 4 splits, it is blind to within-leaf variation.

**Probability calibration**
Decision Tree: probabilities cluster at discrete values (one per leaf — max 16 leaves at depth 4). Random Forest: probabilities spread continuously because each prediction averages 200 trees, each with different leaf boundaries.

**OOB error stabilises early**
OOB error flattens well before 200 trees on this dataset. Diminishing returns kick in around 75–100 trees — useful to know before tuning `n_estimators`.

**Feature importance stability**
`Contract`, `tenure`, and `MonthlyCharges` maintain consistently high importance across all 10 random seeds (low std). Features like `gender` and `PhoneService` show near-zero importance — stable in the sense that they are stably useless.

## Concepts Learned

**Why top-of-tree features matter**
The root node selects the feature with the highest Gini gain across the entire dataset. Features near the top carry global signal — they affect every prediction. Features near the leaves only refine decisions for progressively smaller subsets.

**What Random Forest fixes (and what it doesn't)**
RF solves variance and probability discretisation by averaging across many trees trained on random bootstrap samples. It does not fix: impurity-based bias toward high-cardinality features, under-representation of rare segments (e.g. senior customers with short tenure), or the fundamental limit of Gini-based splits. Boosting (Day 11) addresses the residual errors RF leaves behind.

**Out-of-bag (OOB) error**
Each tree in a Random Forest is trained on ~63% of rows (bootstrap sample). The remaining ~37% — the out-of-bag rows — are used to estimate generalisation error for free, without a separate validation set. `oob_score=True` enables this in sklearn.

## Data Preprocessing Notes

- `TotalCharges` had 11 whitespace-only strings — coerced to NaN and imputed with median
- `customerID` dropped (not a predictive feature)
- Binary columns (`gender`, `Partner`, etc.) → 0/1
- Yes/No/No-service columns → 1/0/0 (treating "No X service" as No)
- Multi-class columns (`Contract`, `InternetService`, `PaymentMethod`) → `LabelEncoder`
- Trees do not require feature scaling
